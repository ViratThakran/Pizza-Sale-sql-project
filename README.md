# Pizza Sales Analysis SQL Project

## Project Overview
This project analyzes pizza restaurant sales data using SQL. The dataset contains customer orders, pizza details, sizes, categories, and pricing information.

## Dataset Tables
1. orders
2. order_details
3. pizzas
4. pizza_types

## Database Relationships
orders -> order_details -> pizzas -> pizza_types

## Project Objectives
- Analyze sales performance
- Measure revenue trends
- Identify best-selling pizzas
- Find peak business hours
- Analyze category performance
- Generate business insights

## Step-by-Step SQL Analysis

### Step 1: Count Total Orders
Purpose: Measure overall business activity.

### Step 2: Calculate Total Revenue
Purpose: Determine total earnings.

### Step 3: Calculate Average Order Value
Purpose: Understand customer spending behavior.

### Step 4: Find Total Pizzas Sold
Purpose: Measure product demand.

### Step 5: Highest Priced Pizza
Purpose: Identify premium products.

### Step 6: Most Popular Pizza Size
Purpose: Understand customer preferences.

### Step 7: Top 10 Best-Selling Pizzas
Purpose: Identify most demanded products.

### Step 8: Revenue by Category
Purpose: Evaluate category performance.

### Step 9: Orders by Hour
Purpose: Discover peak ordering times.

### Step 10: Daily Revenue Trend
Purpose: Monitor day-to-day performance.

### Step 11: Monthly Revenue Trend
Purpose: Analyze seasonal patterns.

### Step 12: Top Revenue Generating Pizzas
Purpose: Identify products generating maximum revenue.

### Step 13: Category Revenue Contribution
Purpose: Understand percentage contribution of each category.
